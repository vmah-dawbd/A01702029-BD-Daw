Cúal es la diferencia entre internet y la World Wide Web? El internet es la conexión entre todos los dispositivos(Computadoras, celulares, tablets) con acceso a internet. La World Wide Web es el conjunto de páginas web asentadas en dicha conexión mencionada en el anterior concepto.[1]
¿Cúal es el propósito de los métodos HTTP: GET, HEAD, POST,PUT,PATCH,DELETE?
  • El método get solicita una representación de un recurso específico. Las peticiones que usan el método GET sólo deben recuperar datos.
  • El método HEAD pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta.
  • El método POST se utiliza para enviar una entidad a un recurso en específico, causando a menudo un cambio en el estado o efectos secundarios en el servidor.
  • El modo PUT reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición.
  • El método DELETE borra un recurso en específico.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué? Se utiliza el método post ya que esta es la encargada de enviar datos a un recurso en específico y cambia el estado de desconocido a usuario registrado de la página.
¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL? Se utiliza get, ya que este método solicita una representación de un recurso, en este caso la ruta de acceso a una página.
Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error? El código 200 representa un ok a nuestra petición. Todo en orden.
¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué? Si. El error 404 indica que la página web ya no existe. Esto se debe a que hubo una actualización y la ruta cambio y el encargado del desarrollo web no actualizo dicha ruta. Es cuando decimos que se cayó el link.
¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué? No es responsabilidad del desarrollador web corregir el sitio web puesto que el error 500 está en el servidor. Esto le toca al encargado de redes de la empresa.[3]

  • tbody - El contenido directo de la tabla.
  • tfoot - Pie de la tabla.
  • th - Titulo de la columna.
  • tr - Contenido de la fila
